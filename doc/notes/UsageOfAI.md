# Notes Marcel
## Infrastruktur
- Ki wurde zum erstellen der Tests für CDK verwendet. Es fiel auf die Tests oft sehr kleinen uMfang hatten und manuell angereichert wurden, um weitere Assertions durchzuführen. Insebsondere bei repitativen Tests kann Ki eine große Hilfe sein, um die Tests zu erstellen. Es ist jedoch wichtig, die Tests sorgfältig zu überprüfen und gegebenenfalls manuell anzureichern, um sicherzustellen, dass sie den Anforderungen entsprechen und eine ausreichende Abdeckung bieten.
- Die nutzung von KI zum erstellen des CDK Code kann nur bedingt empfohlen werden. Häufig halliziniert die KI, aufgrund der Eigenschaft das CDK eine Sprache in einer Programmiersprache ist. Einfache Konstrukte konnte die KI größtenteils selber erstellen, jedoch bei komplexeren Konstrukten hatte die KI syntaktische Fehler implementiert, weil Funktionen zum Beispiel nicht in CDK existierten. Es ist daher wichtig, den von der KI generierten Code sorgfältig zu überprüfen und durch Tests zu validieren.
- KI war eine große Hilfe wenn es darum ging AWS technologien und Umsetzungsmöglichekeiten zu explorieren. Dabei konnte die AI mehrere Mögliochkeiten Aufzeigen und die Vor- und Nachteile der jeweiligen Möglichkeiten erläutern. Wie erwäht war der Schritt zur CDK Implementierung jedoch häufig holperig.


## Allgemein
- KI kann eine große Hilfe sein, um spezifische kleine Fragen zu erlääutern. Beim generieren großer Code Passagen ist jedoch aufgefallen, dass der Lerneffekt deutlich geringer war als bei der manuellen Erstellung von Code. Es ist daher wichtig, die KI als Werkzeug zu nutzen, um spezifische Fragen zu klären, aber auch selbst aktiv zu bleiben und den Code manuell zu erstellen, um den Lerneffekt zu maximieren. KI als helfer oder intelligente Autokomplettierung zu nutzen, kann eine gute Möglichkeit sein, um den Lernprozess zu unterstützen, aber es ist wichtig, die KI nicht als Ersatz für das eigene Lernen und die eigene Entwicklung zu betrachten.
